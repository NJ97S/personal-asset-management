## 1. 해외 서비스 (Personal Finance Management)

### 1.1 서비스별 요약

| 서비스 | 철학 | 가격 (2026) | 타겟 | 핵심 차별점 | 주요 약점 |
|-------|------|-------------|------|------------|----------|
| **YNAB** | Zero-based budgeting ("모든 달러에 역할 부여") | $14.99/월, $109/년 | 훈련된 예산 사용자 | 방법론 중심, 철저한 수동 | 러닝커브 큼, AI/자동화 없음, 투자 추적 미지원 |
| **Monarch Money** | Mint의 후계자 (설계자가 창립) | $14.99/월, $99.99/년 | 일반/커플/가족 | 순자산+투자+예산 통합, 계정 aggregation | 구독료 비쌈, 외국인에겐 연동 제한 |
| **Copilot Money** | "iOS 네이티브 디자인 우선" | $13/월, $95/년 | 애플 생태계 | AI 카테고리 학습 엔진 | iOS/Mac 전용, Android/Windows 미지원 |
| **Rocket Money** | 구독 관리 특화 | 무료+유료 ($6-12/월) | 캐주얼 사용자 | 구독 취소 자동화, 청구서 협상 | PFM 기능은 얕음 |
| **Empower (구 Personal Capital)** | 투자/자산 중심 | 무료 (자산관리 수수료로 수익) | 고자산 투자자 | 포트폴리오 분석, 은퇴 계획 | 대시보드 중심, 예산 기능 약함 |
| **Kubera** | 수동 입력 넷워스 트래커 | $150/년 | 고자산가, 크립토/부동산 보유자 | 모든 자산 (크립토/부동산/NFT) 수동 트래킹 | 비싸고 예산 기능 없음 |
| **Actual Budget** | 오픈소스 YNAB 대안 | 무료 (셀프호스팅) 또는 호스팅 $79/년 | 기술 친화 사용자 | 100% 오픈소스, 로컬 데이터, 자체 호스팅 | 자동 연동 어려움, 기술 지식 필요 |
| **Simplifi (by Quicken)** | 간편 예산 | $5.99/월 | 가벼운 사용자 | 저렴, 지출 트렌드 시각화 | 커스터마이징 제한 |
| **Mint (종료)** | 무료+광고 모델 | 2024-03-23 종료 | - | - | 수익화 실패 → Intuit이 Credit Karma로 통합 |

### 1.2 Mint의 실패가 주는 교훈

- **무료+광고 모델은 지속 불가능**: Intuit은 Mint로 10년 넘게 고전했고 결국 Credit Karma로 흡수
- **광고/추천 중심 수익화는 사용자 경험을 훼손**: 사용자는 스팸성 추천에 피로감
- **데이터 연동 비용은 aggregator(Plaid 등) 구독료로 계속 발생**: B2C 무료로는 감당 어려움
- **교훈**: 유료 구독 모델(YNAB/Monarch)이 장기 생존. 무료 서비스는 개인 개발자 셀프호스팅 오픈소스(Actual Budget)이 유일한 현실 대안

### 1.3 PFM 디자인 패턴 분류

**데이터 입력 축:**
- **자동화 중심 (Mint, Monarch, Copilot)**: 은행/카드 계정 연동, 자동 분류
- **수동 중심 (YNAB, Actual Budget, Kubera)**: 사용자가 직접 입력하며 '의식'하게 만듦
- **하이브리드 (Simplifi, Rocket Money)**: 자동 수집 + 수동 조정

**예산 접근 방식:**
- **Zero-based budgeting (YNAB, EveryDollar, Actual Budget)**: 모든 수입을 미리 카테고리에 배정
- **카테고리 한도 (Simplifi, Monarch)**: 카테고리별 월 한도 설정, 초과 시 경고
- **목표 기반 (Monarch, Rocket Money)**: 저축/투자 목표 중심
- **Envelope 방식 (Goodbudget)**: 봉투처럼 카테고리별 현금 구획

**관점 축:**
- **현금흐름 중심 (YNAB, Simplifi)**: 월별 들어오고 나가는 돈
- **순자산 중심 (Monarch, Kubera, Empower)**: 총 자산 - 부채 트렌드
- **통합 (Monarch)**: 둘 다

### 1.4 공통 Table Stakes 기능

| # | 기능 | 설명 |
|---|-----|------|
| 1 | **거래 내역 기록** | 날짜, 금액, 메모, 카테고리, 계정 |
| 2 | **카테고리 분류** | 식비/교통/주거 등, 계층 구조 지원 |
| 3 | **계정 관리** | 현금/은행/카드/증권 등 여러 계정 |
| 4 | **월별 지출 리포트** | 카테고리별 차트, 전월 비교 |
| 5 | **예산 설정** | 카테고리별 한도 + 경고 |
| 6 | **검색/필터** | 기간/카테고리/금액 필터링 |
| 7 | **반복 거래** | 월세/구독료 등 주기적 항목 |
| 8 | **CSV import/export** | 데이터 주권, 잠김 방지 |

### 1.5 서비스별 차별화 기능 (참고용)

- **YNAB**: 4가지 규칙(Age of Money 등), 과다지출 시 '커버' 조정 UX
- **Monarch**: 다중 사용자(커플/가족), 투자 상세 (종목별 수익률)
- **Copilot**: AI 카테고리 자가 학습, 애플 위젯/Shortcuts 통합
- **Rocket Money**: 구독 자동 감지 + 원클릭 취소
- **Actual Budget**: 로컬 우선, 로컬→서버 sync, 자체호스팅
- **Kubera**: 사망 시 상속자에게 자산 정보 전달 (Beneficiary)

## 2. 국내 서비스

### 2.1 서비스별 요약

| 서비스 | 유형 | 데이터 방식 | 타겟 | 차별점 | 약점 |
|-------|-----|-----------|------|-------|------|
| **뱅크샐러드** | 자산 분석 전문 | 마이데이터 | 자산 관리 관심층 | 가장 촘촘한 자산 연동 (저축은행/새마을금고까지), 카테고리 예산, 건강검진/DNA 등 부가 서비스 | 광고/추천 상품 많음, 앱이 무거움 |
| **토스 (자산)** | 올인원 금융 슈퍼앱 | 마이데이터 | 20-30대 대중 | 가장 쉬운 UI, 금융 전 영역 통합, 간편 송금 연동 | 가계부 기능 상대적으로 얕음 |
| **카카오페이 (자산)** | 생활 중심 금융 | 마이데이터 | 카카오톡 사용자 | 카톡 간편 결제 연동, 생활 결제 중심 | 자산 분석 깊이 부족 |
| **편한가계부** | 전통 수동 가계부 | 수동 + SMS 파싱 | 철저한 기록형 | 단순함, 빠른 입력, 로컬 저장 | 자산 연동 없음, 리포트 약함 |
| **머니매니저 (Realbyte)** | 장수 유료 가계부 | 수동 + 카드 문자 파싱 | 중장년층, 장기 사용자 | 안정성, 다국어, 클라우드 백업 | UI 구식 |
| **브로콜리** | 투자 자산 관리 | 마이데이터 + 증권 | 투자 관심층 | 주식/펀드/크립토 통합 자산 추이 | 지출 관리 기능 약함 |

### 2.2 한국 시장의 구조적 제약

**마이데이터 사업자 인가 필수** — 개인이나 소규모 업체는 은행/카드사 직접 연동 불가능. 뱅크샐러드/토스/카카오페이 같은 기업만 가능.

**개인 개발자가 쓸 수 있는 데이터 수집 수단:**
1. **수동 입력** (가장 안정적)
2. **문자 파싱** (안드로이드 SMS 권한 필요, iOS는 제한적)
3. **이메일 파싱** (카드 명세서, 은행 알림)
4. **CSV import** (은행/카드사 웹에서 다운로드)
5. **스크린샷 OCR** (토스 등에서 내역을 캡처 후 파싱 — 실험적)

### 2.3 국내 사용자의 공통 페인 포인트

- **광고/금융상품 추천 피로감** (뱅크샐러드 공통 불만)
- **중요 데이터를 남의 서버에 맡기는 불안감**
- **앱이 무겁고 슈퍼앱화되어 가계부 기능 찾기 어려움** (토스/카카오페이)
- **전통 가계부는 UI가 옛날스러움** (편한가계부, 머니매니저)
- **한국어 기반 카테고리 자동 분류 정확도** (은행마다 거래 명칭이 제각각)

## 3. 개인 개발자 / 개인용 프로젝트 관점의 시사점

### 3.1 현실적 기술 제약
- ❌ 마이데이터 연동 불가
- ❌ 은행/카드 OAuth 연동 불가 (Plaid 같은 것 없음)
- ✅ 수동 입력 + CSV 가져오기 + 문자 파싱 (안드로이드)이 실질적 선택지
- ✅ PWA로 데스크탑+모바일 동시 지원 가능 (앱스토어 배포 없이)

### 3.2 "개인용"이라 할 수 있는 것
- 내 취향에 맞는 카테고리 체계 자유롭게 설계
- 데이터 주권 (로컬 DB 또는 내가 제어하는 서버)
- 광고/추천 없는 깨끗한 UI
- 독특한 개인 룰 구현 (예: "투자는 비용이 아니라 자산 이동으로 처리")
- 장기 보관 (YNAB도 5년 이상 데이터는 요약)
- 세금 신고용 지출 분류 등 나만의 니즈

### 3.3 레퍼런스 삼을 만한 서비스
- **Actual Budget** (오픈소스, 로컬 우선, 동기화) — 가장 가까운 레퍼런스
- **YNAB의 방법론** — zero-based budgeting 아이디어
- **Copilot의 UX 미니멀리즘** — 디자인 철학
- **뱅크샐러드의 카테고리 체계** — 한국 지출 패턴 반영

### 3.4 피해야 할 함정
- Mint 식 "모든 기능 다" → 유지보수 지옥
- Plaid/마이데이터 흉내 시도 → 불가능
- 과도한 시각화 → 쓰지 않는 기능이 됨
- 복잡한 예산 시스템 → 첫 달만 쓰고 포기
- 네이티브 모바일 앱 (RN/Flutter) → 개인 개발자에겐 관리 부담 큼, PWA가 유리
